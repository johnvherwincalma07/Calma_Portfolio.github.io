II. GIVE 5 COMMANDS YOU MASTER AFTER OUR ACTIVITY

1. chmod (symbolic mode)
Purpose: Change file permissions using letters (u, g, o)
Example: chmod u+x file.txt
Result: Execute permission is added to the user
Meaning: Allows control of permissions using symbols

2. chmod (octal mode)
Purpose: Set permissions using numeric values
Example: chmod 755 file.txt
Result: rwxr-xr-x permissions applied
Meaning: Uses numbers to quickly assign permissions

3. ls -l
Purpose: View file permissions and details
Example: ls -l
Result: -rwxr-xr-x file.txt
Meaning: Shows who can read, write, or execute the file

4. tar (archive with compression)
Purpose: Archive and compress files into one file
Example: tar -czvf archive.tar.gz folder/
Result: archive.tar.gz is created
Meaning: Combines and compresses files for easier storage or transfer

5. tar (extract compressed archive)
Purpose: Extract archived and compressed files
Example: tar -xzvf archive.tar.gz
Result: Files are extracted from archive
Meaning: Restores files from compressed archive

III. REFLECTION AFTER OUR SESSION

After our session, I learned how to manage file access permissions in Kali Linux. I understood how to use both symbolic and octal modes in chmod to control file access. The ls -l command helped me clearly see permission settings for each file. In addition, I also learned how to archive and compress files using tar, which is useful for organizing and transferring files efficiently. Overall, this activity improved my understanding of both file security and file management in Linux systems.

II. GIVE 5 LESSONS YOU MASTER AFTER OUR ACTIVITY

1. First Come First Serve (FCFS)
Purpose: Executes processes based on arrival time
Example: P1 → P2 → P3
Result: Processes run in order of arrival
Meaning: Simple scheduling but may cause long waiting time

2. Shortest Job First (SJF)
Purpose: Executes the shortest process first
Example: Process with smallest burst time runs first
Result: Faster average waiting time
Meaning: Efficient but needs burst time estimation

3. Round Robin (RR)
Purpose: Each process gets equal time (time quantum)
Example: P1 → P2 → P3 → repeat
Result: Fair CPU sharing
Meaning: Prevents starvation

4. Priority Scheduling
Purpose: Executes based on priority level
Example: High priority runs first
Result: Important tasks execute earlier
Meaning: Low priority may wait longer

5. Gantt Chart
Purpose: Visual representation of process execution
Example: | P1 | P2 | P3 |
Result: Shows execution order and time
Meaning: Helps analyze scheduling performance

III. REFLECTION AFTER OUR SESSION

This activity helped me understand different CPU scheduling algorithms. I learned how processes are arranged and executed using methods like FCFS, SJF, Round Robin, and Priority Scheduling. Creating a Gantt Chart helped me visualize the order of execution and compute waiting and turnaround times. Overall, I realized how important scheduling is in improving system performance and ensuring fairness among processes.
